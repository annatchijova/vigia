Un agente que no confunde datos con confianza.

El flujo no depende de una única fuente ni de una pantalla perfecta. Cada ciclo obtiene lecturas, compara inconsistencias, aplica una corrección determinista, evalúa umbrales y deja registro del resultado.

01 / OBTENER

Lecturas de campo

Consulta niveles disponibles de Prefectura Naval Argentina y del INA; también admite un sensor local cuando el dispositivo lo tiene.

02 / CONTRASTAR

La discrepancia es una señal

Busca diferencias relevantes entre fuentes y tendencias opuestas, en vez de ocultarlas como si fueran una única verdad.

03 / CORREGIR

Reglas explícitas

Ante una contradicción, ajusta la prioridad de fuente o marca el nodo para revisión humana. La decisión queda documentada.

04 / ALERTAR

Conserva lo urgente

Evalúa umbrales de alerta y evacuación. Si no hay red, preserva las lecturas y encola la alerta para su tratamiento posterior.

Una contradicción entre PNA, INA, simulación o una tendencia observada no se disfraza de certeza: se registra, se trata con una regla y puede requerir revisión humana.

La alerta es sólo el comienzo de la evidencia.

Cada lectura y cada acción del agente se incorporan a un audit trail. El objetivo no es sólo enviar un mensaje: es poder reconstruir qué observó el sistema, qué fuente usó y qué regla aplicó.

Audit trail hidrológico

Las entradas incluyen secuencia, fecha, caso, herramienta, resumen de entrada y salida, iteración y una huella SHA-256 de la entrada. Los payloads crudos de las fuentes también se hashean.

Preparado para condiciones imperfectas.

La interfaz no es un tablero de emergencia ni reemplaza a autoridades, sensores certificados o protocolos de protección civil. Es una capa de monitoreo, trazabilidad y notificación para acompañar decisiones humanas responsables.

Cuando los canales responden

El agente obtiene lecturas, actualiza nodos, detecta discrepancias, evalúa los umbrales configurados y puede enviar una notificación por Telegram o WhatsApp si esos canales fueron configurados.
